## Why

The project's own roadmap gates inline (blocking) enforcement on knowing the
classifier's real false-positive rate — and that rate has never been measured.
Separately, the D54 audit noted peer-UEBA is an admitted "toy z-score" with two
concrete flaws. Both are detection-QUALITY gaps: the pipeline works, but nothing
says how well it discriminates. This measures the classifier honestly and hardens
peer-UEBA against its two known failure modes.

## What Changes

**Part A — measure classifier detection quality.** A labeled corpus and a
measurement harness compute and RECORD precision / recall / false-positive rate:
- Positives: valid PII (real valid CPFs and Luhn-valid cards, generated by
  computing their check digits) embedded in realistic sentences — must be detected.
- Negatives: clean text (prose, code, logs) PLUS adversarial near-misses (an
  11-digit number whose CPF check digits are wrong, a 16-digit number failing
  Luhn, SSN-shaped numbers) — must NOT be flagged.
- Asserted honestly: the checksum-backed detectors (CPF, card) have ~zero FP on
  near-misses; recall is high on valid PII; and the SSN detector — structural
  only, no checksum — has a materially HIGHER FP rate on SSN-shaped numbers,
  surfaced as the measured reason its confidence is capped low (D4/D5). The
  numbers become a regression guard: weakening a checksum makes FP jump.

**Part B — harden peer-UEBA past the toy z-score.** Fix the two known flaws:
- SELF-CONTAMINATION: the subject's own activity is currently in the mean/stddev
  it is compared against. Fix with a leave-one-out peer baseline (exclude the
  subject under test).
- NO DECAY: cumulative never-decayed counts let a benign-but-busy subject climb
  into "outlier" over time. Fix with time decay so old activity fades.
- Each fix proven against its specific flaw. Still a statistical baseline (not a
  trained detector), server-side, off by default (D23), observe-only (D54); the
  Analyzer API (Observe/ContextFor/Resolver) and context_version (D53) unchanged.

### Modified Capabilities
- `pattern-classifier`: add a measured detection-quality floor (FP/precision/recall
  against a labeled corpus incl. adversarial near-misses).
- `peer-ueba`: the peer baseline excludes the subject under test and decays old
  activity, fixing self-contamination and unbounded drift.

## Impact

- New code: a classifier corpus + measurement test (generated valid/near-miss PII,
  committed realistic clean text); peer-UEBA leave-one-out + time-decay in
  `internal/analytics/peerueba`. Docs (new D-number recording the measured floor
  and the model hardening).
- No API breakage: the Analyzer's public surface and the classifier's detector
  contract are unchanged; only internals and a new measurement guard.
- Honest bounds preserved: the classifier is patterns+checksums (D5), peer-UEBA is
  a baseline not an ML detector, off by default, observe-only. The measurement is
  on a SYNTHETIC corpus — it validates the checksum FP defense and discrimination,
  NOT real-world base rates, which still need live dogfood (T-015).

## Context

`internal/classify/detectors.go` scans text with format regexes gated by
validators: CPF (two check digits mod 11, conf 0.95), card (Luhn, 0.90), SSN
(structural only, no checksum, 0.60), email (regex). The checksums ARE the FP
defense, but nothing measures whether they hold against adversarial near-misses.
`internal/analytics/peerueba` keeps cumulative per-subject counts and scores a
z-score of a subject against the population mean/stddev — including the subject
itself, and with no time decay.

## Goals / Non-Goals

**Goals:**
- A recorded, honest FP/precision/recall for the classifier on a labeled corpus
  that includes adversarial near-misses; a regression guard on the checksum FP
  defense.
- peer-UEBA: a leave-one-out peer baseline and time decay, each proven to fix its
  specific flaw, with the public API unchanged.

**Non-Goals:**
- Real-world base rates or a trained/ML detector — the corpus is synthetic; it
  validates discrimination and the checksum defense, not field accuracy (that is
  dogfood, T-015).
- New detector types, or changing detector confidences/contracts.
- Changing peer-UEBA's shape (still a statistical baseline, off by default,
  observe-only) or its Observe/ContextFor/Resolver signatures.

## Decisions

**Part A: generate valid PII and near-misses deterministically; commit realistic
clean text.** Valid CPFs and Luhn-valid cards are generated by computing their
check digits (so positives are genuinely valid, not hand-picked); near-misses are
a valid number with one digit flipped so the checksum fails — the sharpest FP
test. Clean prose/code/log text is committed as fixtures (realistic negatives).
The harness scans each corpus item and tallies detections into a confusion matrix
per detector, then asserts:
- CPF and card FP rate on near-misses == 0 (a failed checksum must never flag);
- recall on valid CPF/card >= a high floor;
- SSN FP rate on SSN-shaped numbers is recorded and allowed to be materially
  higher (no checksum) — the test PRINTS the numbers and asserts the ordering
  (SSN FP > CPF FP), making the "weak by construction" claim (D4/D5) a measured
  fact, not just a comment.

**A regression guard, not a fixed magic number.** The assertions are floors/
ceilings tied to the checksum property (near-miss FP == 0), not brittle exact
counts, so the test is a guard against a real regression (a weakened checksum),
not corpus-churn noise. Measured numbers are logged for the record.

**Part B: leave-one-out baseline.** `ContextFor(subject)` computes mean/std over
the OTHER subjects' counts, not all counts — so the subject never contaminates
the baseline it is judged against. This makes an outlier score MORE clearly
anomalous (its own large count no longer inflates the mean/std it is compared to).
Requires >= 2 OTHER subjects to have peers (was >= 2 total); documented.

**Part B: time decay.** Counts decay toward zero as time passes, so a steady rate
of activity reaches a STEADY level rather than climbing forever. Implemented as an
exponential decay applied on read/update against a half-life, using a monotonic
tick supplied to Observe (or an injected clock) — NOT wall-clock inside the pure
analyzer, to keep it testable and deterministic. A steady-but-busy subject then
sits at a bounded level near its peers instead of drifting into "outlier"; a
genuine burst still spikes above the decayed peer level.

**Clock injection for determinism.** The analyzer takes an optional time source
(default a real monotonic clock); tests drive it explicitly. This keeps decay
deterministic and avoids the `Date.now`-style nondeterminism that has bitten
before.

## Risks / Trade-offs

- **Synthetic corpus ≠ field accuracy.** The measured FP/recall validate the
  checksum defense and discrimination on constructed inputs; real documents have
  different base rates and messier text. Stated plainly — this retires the
  "checksums untested" risk, not the "unknown real FP rate" risk, which still
  needs dogfood (T-015).
- **Decay adds a parameter (half-life).** A wrong half-life mis-tunes sensitivity
  (too fast → forgets real accumulation; too slow → still drifts). Chosen with a
  documented default; it is a knob, and the test pins the qualitative behavior
  (steady subject bounded, burst still detected), not a magic sensitivity number.
- **Leave-one-out with a tiny population is noisy.** With few peers the excluded
  baseline has high variance. The >= 2-other-subjects floor and the nil-Context
  return for too-small populations keep it from emitting noise, unchanged in
  spirit from the current small-population guard.
- **Still not a trained detector.** Both parts harden a statistical baseline; they
  do not make peer-UEBA an ML system or the classifier an NER model. The honest
  framing (D4/D5, D54) is preserved.
